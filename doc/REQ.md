# 專案需求文件

## 核心問題摘要

### 1. 為什麼要做這個專題?
因為我們要做遊戲的攻略。

### 2. 使用者是誰?
使用者是會玩這遊戲的玩家。

### 3. 他們希望能做什麼?
希望能夠在這網站可以查到遊戲攻略。

### 4. 成功的指標是什麼?
玩家們看到遊戲攻略而變得更強。

### 5. 哪些不在專案範圍內?
我們專注於「怎麼玩」，而不是「好不好玩」。我們不是新聞網站，不會即時追蹤官方的芝麻小事（除非是重大改版，會影響攻略內容）。

---

## 詳細需求說明

### 1. 專題名稱 / Project Title / 프로젝트명
《Peak》攀登攻略站

### 2. 專案目的 / Purpose / 목적
建立一個專為《Peak》設計的攻略網站，提供最完整的攀登路線圖、地圖指引與收集品位置，幫助玩家克服困難地形、順利登頂，並探索遊戲中所有的隱藏區域。

### 3. 主要使用者 / Target Users / 주요 사용자
*   **新手登山客：** 剛開始玩，需要了解基本攀登技巧、耐力管理，並尋找「最簡單」的登頂路線。
*   **迷路的探索者：** 卡在某個懸崖或區域，找不到正確的路，需要特定區域的路線指引。
*   **完美主義收藏家：** 追求全收集、全成就，需要知道所有隱藏道具、風景點或彩蛋的精確位置。

### 4. 網站主要功能 / Key Features / 주요 기능
| 分類 | 功能描述 | 優先級 |
| :--- | :--- | :--- |
| 互動地圖 | 提供全區域地圖，可篩選標示重要地標、收集品、捷徑、休息點的位置。 | Must |
| 主線登頂攻略 | 提供「登頂」的主要路線圖文指引，包含關鍵的攀爬點與耐力管理技巧。 | Must |
| 全收集攻略 | 條列所有收集品（例如：羽毛、旗幟、信件等）的詳細位置與取得方式。 | Must |
| 搜尋功能 | 讓使用者能快速搜尋「地標名稱」、「區域名稱」或「收集品名稱」。 | Must |
| 攀登機制解說 | 遊戲核心的攀登、耐力、天氣影響、特殊裝備（如鉤爪）等系統。 | Should |
| 隱藏區域/支線 | 整理主線外的隱藏山峰、洞穴或NPC任務的探索指南。 | Should |
| 成就/獎盃 | 提供所有成就的解鎖條件與技巧。 | Could |

**優先級說明：**
*   **Must:** 必須完成的核心功能。對於爬山遊戲來說，「地圖」和「收集」是核心。
*   **Should:** 非常重要的功能，能大幅提升網站體驗。
*   **Could:** 如果時間和資源允許，可以做的加分項目。

### 5. 成功指標 / Success Metrics / 성공 지표
*   網站能正常且快速地顯示於電腦與手機瀏覽器 (RWD)。
*   玩家透過搜尋引擎（如Google）搜尋「Peak 收集品」、「Peak 怎麼爬」時能找到我們的網站。
*   高互動率： 玩家在「互動地圖」頁面的停留時間長、篩選功能使用率高。
*   （長期）玩家在留言區回報攻略的準確性（例如：「感謝！終於找到了！」）。

### 6. 非範圍內容 / Out of Scope / 범위 밖
*   不含會員登入、註冊系統。
*   不含金流功能（如付費攻略、商城）。
*   （初期）不製作影音攻略（製作成本高，先專注圖文與地圖）。
*   不提供遊戲新聞、評論（專注於「攻略」）。
*   不開發獨立的手機 App。
*   不包含 Speedrun (競速) 路線，除非這是遊戲的核心玩法（先專注探索）。

## 宣傳活動企劃

### 專題名稱: 《Peak》攻略站宣傳活動企劃 (《Peak》Guide Promotion Campaign)
### Project Goal: 提升《Peak》攻略站的能見度與使用者流量
### 主要使用者:
*   潛在玩家 (學生/一般玩家): 正在玩《Peak》或考慮玩《Peak》的玩家，需要攻略來幫助他們。
*   遊戲社群管理員/影響者: 幫助擴散攻略站資訊，或引用我們的攻略內容。

### 📈 KPI (關鍵績效指標):
*   攻略站點擊率 (CTR) > 15%: 從宣傳素材（社群貼文、論壇連結等）點擊進入攻略站的比例。
*   網站活躍使用者 (Active Users) 每週成長 20%: 衡量新進使用者與回訪使用者數量。
*   特定攻略頁面（例如：某個難點的Boss/路線）的分享次數 > 100 次: 衡量內容的實用性與玩家分享意願。
*   《Peak》攻略站的搜尋引擎排名 (SEO Ranking) 進入前 5 名: 針對「Peak 攻略」、「Peak 收集品」等關鍵字。

### 🧩 功能需求 (用 AI 工具強化宣傳素材製作):
**宣傳影片自動剪輯 (使用 Veo 2 或 Gemini Image):**
*   目標： 快速製作吸睛的短影音，展示攻略站的核心價值與遊戲中的精彩畫面。
*   應用方式：
    *   Veo 2： 將預錄好的《Peak》遊戲精彩畫面（例如：成功登頂、發現隱藏收集品、克服困難地形）的片段匯入。利用 Veo 2 的自動剪輯功能，快速生成不同時長（15秒、30秒）的宣傳片。可設定主題（如「困難挑戰」、「探索樂趣」），讓 Veo 2 自動配樂、加入轉場特效。
    *   Gemini Image： 生成用於影片開頭/結尾的視覺圖卡 (Title Card/End Card)，例如：「你卡關了嗎？《Peak》攻略站幫你解惑！」或是網站的 QR Code 導流圖。同時可以生成影片的封面縮圖 (Thumbnail)，吸引點擊。
*   內容範例：
    *   「突破難關」系列：剪輯玩家在攻略站幫助下成功克服某個難點的畫面。
    *   「收集大師」系列：展示攻略站如何引導玩家找到所有隱藏收集品。
    *   「美景探索」系列：呈現遊戲中壯麗的山景，並提示攻略站提供詳細探索路線。

**社群貼文生成 (搭配 Gemini Image 強化視覺效果):**
*   目標： 根據不同社群平台（FB、IG、X/Twitter、PTT/巴哈姆特）的特性，快速生成吸引人的文字與圖片。
*   應用方式：
    *   Gemini (文字部分)： 根據預設的攻略站連結、重點功能（互動地圖、全收集攻略）和 CTA (Call To Action)，自動生成多個版本的社群貼文文案。例如：
        *   FB 長篇文案：詳細介紹攻略站的特色。
        *   X/Twitter 短文案：加上 #Peak #遊戲攻略 #爬山 等Hashtag。
        *   巴哈姆特/PTT 討論區：模擬玩家口吻發布求助文或推薦文。
    *   Gemini Image (圖片部分)：
        *   生成符合《Peak》遊戲風格的精美宣傳圖，例如結合遊戲畫面與攻略站介面的概念圖。
        *   為不同的社群貼文生成客製化配圖，例如「挑戰這個 Boss 你需要...」的圖文並茂貼文。
        *   製作資訊圖表 (Infographic)，簡潔說明攻略站的亮點功能。

**活動頁面設計 (網站導流):**
*   目標： 確保從宣傳素材點擊進來的用戶，能清晰地看到攻略站的核心價值，並找到他們需要的資訊。
*   應用方式：
    *   Landing Page (登陸頁面): 針對不同的宣傳管道，可以設計略有差異的登陸頁面。例如，如果是為了宣傳「全收集攻略」，則該登陸頁面直接引導至全收集攻略的區塊。
    *   主要 Call To Action (CTA): 明確的按鈕或連結，引導用戶進入「互動地圖」、「主線攻略」或「收集品列表」。
    *   視覺設計： 頁面排版應簡潔明瞭，使用《Peak》的遊戲元素和色彩風格，提升沉浸感。
