# 專案需求規格書 (15小時原型開發版)

---

### **核心問題摘要**

1.  **為什麼要做這個專題?**
    因為我們要做遊戲的攻略。

2.  **使用者是誰?**
    使用者是會玩這遊戲的玩家。

3.  **他們希望能做什麼?**
    希望能夠在這網站可以查到遊戲攻略。

4.  **成功的指標是什麼?**
    玩家們看到遊戲攻略而變得更強。

5.  **哪些不在專案範圍內?**
    我們專注於「怎麼玩」，而不是「好不好玩」。我們不是新聞網站，不會即時追蹤官方的芝麻小事（除非是重大改版，會影響攻略內容）。

---

### **1. 專題簡介 (Project Introduction)**

本文件定義的是一個在 **15 小時內**開發完成的**網站原型**。

此原型旨在為遊戲《Peak》的玩家提供一個**概念驗證 (Proof of Concept)** 的攻略站。它將展示核心功能，但內容和技術實現都將被**極度簡化**，以確保在有限時間內的可行性。

專案目標是建立一個可以操作、可以展示的最小可行產品 (MVP)，而不是一個功能完整的網站。

---

### **2. 使用者角色 (User Roles)**

原型將服務於以下核心使用者，但在功能上僅滿足其最基本的需求：

| 使用者類型 | 核心痛點 (他們為何而來) | 在原型中的體驗 (我們提供什麼) |
| :--- | :--- | :--- |
| **新手登山客** | **挫折與無助感**：遊戲初期難度過高，容易因迷路和不熟悉機制而放棄。 | 能透過靜態圖文攻略，了解**第一個主線任務**的玩法。 |
| **迷路的探索者** | **時間浪費與煩躁感**：在特定地點卡關，進度完全停滯，只想快速找到解答。 | 能在**僅包含初始區域**的地圖上，點擊固定的標示點查看資訊。 |
| **完美主義收藏家**| **擔心錯過與低效率**：害怕遺漏隱藏要素，且地毯式搜索耗時又枯燥。 | 能在地圖上看到**一種類型**的收藏品在初始區域的位置。 |

---

### **3. 使用情境 (Scenarios)**

| 情境 | 使用者目標 | 系統如何協助 (原型) |
| :--- | :--- | :--- |
| **情境一：新手初試身手** | 新手玩家想知道遊戲一開始該怎麼玩。 | 玩家點擊「主線攻略」頁籤，看到一個靜態頁面，上面有文字和圖片，指導他完成第一個任務。 |
| **情境二：探索初始區域** | 玩家想知道初始區域「寧靜森林」中有什麼。 | 玩家點擊「地圖」頁籤，看到一張「寧靜森林」的地圖圖片。他可以點擊圖上預設的 1-2 個圖標，彈出簡單的文字說明，如「這裡是第一個休息點」。 |

---

### **4. 功能需求 (Functional Requirements)**

此原型的所有功能均為 **Must** 等級，且實現方式將被大幅簡化。

| 功能名稱 | 功能描述 (簡化版) |
| :--- | :--- |
| **簡化版互動地圖** | 1. 顯示一張**固定的、不可縮放**的遊戲初始區域地圖圖片。<br>2. 地圖上會有 2-3 個**預先設定好**的點位。<br>3. 點擊這些點位，會用 JavaScript 彈出一個包含簡短文字的提示框。 |
| **靜態圖文攻略** | 1. 一個獨立的 HTML 頁面或區塊。<br>2. 頁面內包含**寫死的文字**和 **1-2 張靜態圖片**，用以說明遊戲第一個任務的流程。 |
| **基礎頁面導覽** | 1. 一個簡單的頂部導覽列。<br>2. 提供兩個連結，分別用於切換「地圖」和「攻略」頁面/區塊。 |

---

### **5. 非功能需求 (Non-functional Requirements)**

| 類別 | 需求描述 |
| :--- | :--- |
| **效能 (Performance)** | 作為一個純靜態網站，頁面應能快速載入。 |
| **易用性 (Usability)** | 介面佈局清晰，使用者能輕易理解如何在地圖和攻略頁之間切換。 |
| **瀏覽器相容性** | 能在最新版的 Chrome 或 Edge 瀏覽器上正常顯示即可。 |
| **響應式設計 (RWD)** | 桌面瀏覽器優先。在手機上能基本瀏覽，不要求完美適配。 |
| **安全性 (Security)**| 純靜態網站，無後端邏輯與資料庫，暫不考慮複雜的安全性需求。 |

---

### **6. MVP (最小可行產品) 功能**

為了確保在 15 小時內完成，此專案的最小成功標準是交付以下三個核心功能。其他所有部分（如更精美的樣式、更多的內容）均為次要目標。

1.  **一個簡化版互動地圖頁面**：
    *   能顯示一張固定的地圖圖片。
    *   圖上有至少 2 個可點擊的點位，點擊後能彈出文字說明。

2.  **一個靜態圖文攻略頁面**：
    *   能顯示文字和至少 1 張圖片，用來說明一個任務或一個概念。

3.  **基礎的頁面導覽**：
    *   提供一個導覽列，讓使用者可以在上述兩個頁面之間切換。

---
**範圍之外 (Out of Scope):**
*   任何後端功能 (資料庫、API、使用者登入)。
*   可任意縮放、平移的動態地圖。
*   完整的遊戲內容或攻略。
*   搜尋功能。
*   複雜的 CSS 動畫或視覺效果。
