此章節定義兩個核心頁面的 UI 元件，將非功能需求（如 RWD）具體化為設計約束。

#### 5.1 簡化版互動地圖頁面 (`map.html`)

| 元件名稱 | 類型 | 內容或用途 | 設計約束（與 REQ NFR 追溯） |
| :--- | :--- | :--- | :--- |
| **主容器** | HTML 結構 | 整個頁面的主要佈局。 | **RWD 約束**：必須優先設計桌面瀏覽器佈局，行動裝置上不要求完美適配。 |
| **頂部導覽列** | 連結 | 提供 `map` 和 `guide` 的切換功能。 | 必須佈局清晰，滿足易用性需求。 |
| **地圖圖片** | 靜態 `<img>` | 遊戲初始區域的固定圖片。 | **不可縮放、不可平移**（符合 REQ **Out of Scope**）。 |
| **點擊圖標** | 靜態 `<div>` | 2-3 個預設且**硬編碼座標**的圖標。 | **數量必須符合 REQ 限制**（至少 2 個）。必須使用 CSS 絕對定位。 |
| **提示彈窗 (Modal)** | 自定義 `<div>` | 點擊圖標後彈出的純文字提示框。 | 必須採用 **CSS Modal** 結構，而不是瀏覽器原生彈窗。 |
