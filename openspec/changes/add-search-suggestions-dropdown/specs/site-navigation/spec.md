# site-navigation Specification

## MODIFIED Requirements

### Requirement: 一致性的頁面導覽

應用程式的所有主要頁面都 MUST (必須) 提供一個導覽列。導覽列應包含「遊戲資訊」、「主線攻略」、「互動地圖」連結，以及一個搜尋框。

#### Scenario: 從遊戲資訊頁導覽至主線攻略頁

**Given** 使用者正在「遊戲資訊」頁面 (`index.html`)。
**When** 使用者點擊導覽列中的「主線攻略」連結。
**Then** 應用程式將顯示「主線攻略」頁面 (`main-quest.html`)。

#### Scenario: 從主線攻略頁導覽至互動地圖頁

**Given** 使用者正在「主線攻略」頁面 (`main-quest.html`)。
**When** 使用者點擊導覽列中的「互動地圖」連結。
**Then** 應用程式將顯示「互動地圖」頁面 (`map.html`)。

## ADDED Requirements

### Requirement: 搜尋框提供即時建議

當使用者在搜尋框中輸入文字時，MUST (必須) 顯示一個下拉式建議清單，其中包含頁面上與查詢相關的內容區段連結。

#### Scenario: 輸入查詢時顯示建議清單

**Given** 使用者位於 `index.html` 頁面。
**When** 使用者在搜尋框中輸入「狀態」。
**Then** 一個下拉式清單出現，其中應包含指向標題為「生存機制與狀態效果」和「負面狀態」等區段的連結。

#### Scenario: 點擊建議項目可導覽至對應區段

**Given** 搜尋建議清單可見，並包含「負面狀態」項目。
**When** 使用者點擊「負面狀態」建議項目。
**Then** 頁面應平滑滾動至 ID 為「負面狀態」的區段，且建議清單應被隱藏。

#### Scenario: 點擊頁面其他地方可隱藏建議清單

**Given** 搜尋建議清單可見。
**When** 使用者點擊頁面主要內容區域 (非建議清單或搜尋框)。
**Then** 建議清單應被隱藏。

#### Scenario: 查詢無結果時顯示提示

**Given** 使用者位於 `index.html` 頁面。
**When** 使用者在搜尋框中輸入「一個不存在的查詢詞」。
**Then** 下拉式清單應顯示一則訊息，例如「找不到結果」。
